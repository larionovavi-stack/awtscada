Overview системы

BitSCADA — браузерная SCADA-система industrialого уровня для dispatchского controlия и сбора данных. Process visualization, сбор данных с контроллеров, controlие исfullительными механизмами, archiving и alertие.

Состав поставки
FileSizeAssignment
SCADA_IEC61850_v8.html~240 КБClientская часть (SCADA)
SCADA_Gateway.exe~25 МБServerная часть (Gateway + Emulator)

Installation additionalых программ не требуется. Connection к интернету не требуется.

⚡
99% собственного кода. External dependencies — only React 18 и Babel (загружаются с CDN). Offline-сборка fullостью автономна.

Installation

📌
Installation не требуется. Скопируйте два файла в любую папку.
Download BitSCADA_Demo.zip и распакуйте в любую папку, наatмер C:\BitSCADA\
The folder must contain: SCADA_IEC61850_v8.html (or offline-версия), SCADA_Gateway.exe и демонстрационный проект .json
Installation finishена. Права администратора не требуются.

Auto-load Gateway (рекомендуется)

For автоматического launchа Gateway at enablении компьютера:

# Open папку автоloadи
Win+R → shell:startup → Enter
# Insert shortcut SCADA_Gateway.exe

Quick Start

Run Gateway — двойной клик на SCADA_Gateway.exe. Wait for сообщения All servers running! в консоли. Window не закрывайте.
Open SCADA — двойной клик на SCADA_IEC61850_v8.html. Browser will open automatically.
Проверьте connectие — в заголовке должна быть зелёная pointа GW. In console Gateway: Config received from SCADA.
Run Runtime — нажмите buttonу Runtime в заголовке. Instruments покажут changяющиеся valueия от эмулятора.
✅
If atборы показывают данные — система operationет! Dashboard Gateway accessен на http://localhost:8088

Соединения (провода)

Соединения создаются кликом на выходной порт блока (справа), затем кликом на входной порт другого блока (слева). Провода рисуются как Bezier-кривые.

  • Portы входов — слева блока
  • Portы выходов — справа блока
  • Deletion провода — выделить + Delete

IEC 61850 (MMS / GOOSE / SV)

MMS :102

Read/writь переменных IED. Control и мониторинг.

GOOSE multicast

Fast event transfer (4 мс). Protective автоматика.

SV multicast

Instant valueия токов трёх фаз. Цифровые подстанции.

OPC UA

OPC UA (IEC 62541) — универсальный access к данным. Gateway launchает сервер на opc.tcp://hostname:4840.

Settings: SCADA → Settings → OPC UA → endpoint, порт, режим withoutопасности.

Modbus TCP

Стандартный industrialый protocol. Gateway: порт 502 (or 1502 если busy). Поддерживает functioи: FC 3 (Read Holding), FC 4 (Read Input), FC 6 (Write Single), FC 16 (Write Multiple).

OPC DA / InTouch / WinCC

OPC DA Classic via DCOM. Требуется Windows. Support Kepware, Wonderware InTouch (Wonderware.InTouch.OPCServer), Siemens WinCC (OPCServer.WinCC).

Устранение неполадок

SCADA не подkeyается к Gateway

Проверьте что SCADA_Gateway.exe запущен (lineа All servers running!). Убедитесь что порт 8089 не блокируется антивирусом.

Gateway: «asyncua not found»

Нормальная ситуация для автономной поставки. Gateway operationет в режиме эмуляции — все данные генерируются встроенным эмулятором.

Instruments в Runtime не показывают valueий

Проверьте наличие зелёной pointи GW в заголовке. If отсутствует — Gateway не подkeyён. Click Runtime повторно after launchа Gateway.

Project не сохраняется

localStorage переfullен. Удалите неиспользуемые проекты or экспортируйте в JSON.

Security

  • Trial-режим: 2 часа работы с anti-tamper защитой
  • Целостность: SHA256 хэш JSX-кода, checkа at loadе
  • Base64: исходный код закодирован внутри HTML
  • Protection редактора: logин/пароль в settingах
  • Роли: Operator / Engineer / Admin
  • Пароли: SHA256 хэш в SQLite

Licensing

Built-in licensing without activation servers. Each сборка содержит:

  • Build ID — 16-symbolьный SHA256 хэш
  • License Key — 4 группы по 8 symbolов (MD5)
  • Integrity Hash — SHA256 контрольная сумма
