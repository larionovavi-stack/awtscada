Обзор системы

BitSCADA — браузерная SCADA-система промышленного уровня для диспетчерского управления и сбора данных. Визуализация технологических процессов, сбор данных с контроллеров, управление исполнительными механизмами, архивирование и оповещение.

Состав поставки
ФайлРазмерНазначение
SCADA_IEC61850_v8.html~240 КБКлиентская часть (SCADA)
SCADA_Gateway.exe~25 МБСерверная часть (Gateway + Emulator)

Установка дополнительных программ не требуется. Подключение к интернету не требуется.

⚡
99% собственного кода. Внешние зависимости — только React 18 и Babel (загружаются с CDN). Offline-сборка полностью автономна.

Установка

📌
Установка не требуется. Скопируйте два файла в любую папку.
Скачайте BitSCADA_Demo.zip и распакуйте в любую папку, например C:\BitSCADA\
В папке должны быть: SCADA_IEC61850_v8.html (или offline-версия), SCADA_Gateway.exe и демонстрационный проект .json
Установка завершена. Права администратора не требуются.

Автозагрузка Gateway (рекомендуется)

Для автоматического запуска Gateway при включении компьютера:

# Откройте папку автозагрузки
Win+R → shell:startup → Enter
# Вставьте ярлык SCADA_Gateway.exe

Быстрый старт

Запустите Gateway — двойной клик на SCADA_Gateway.exe. Дождитесь сообщения All servers running! в консоли. Окно не закрывайте.
Откройте SCADA — двойной клик на SCADA_IEC61850_v8.html. Браузер откроется автоматически.
Проверьте подключение — в заголовке должна быть зелёная точка GW. В консоли Gateway: Config received from SCADA.
Запустите Runtime — нажмите кнопку Runtime в заголовке. Приборы покажут изменяющиеся значения от эмулятора.
✅
Если приборы показывают данные — система работает! Dashboard Gateway доступен на http://localhost:8088

Соединения (провода)

Соединения создаются кликом на выходной порт блока (справа), затем кликом на входной порт другого блока (слева). Провода рисуются как Bezier-кривые.

  • Порты входов — слева блока
  • Порты выходов — справа блока
  • Удаление провода — выделить + Delete

Многостраничность

Проект может содержать основную страницу и произвольное количество дополнительных:

  • Кнопка + в заголовке — добавить страницу
  • Вкладки — переключение между страницами
  • Каждая страница имеет свой набор блоков и фигур

Попапы

Попапы — страницы, открывающиеся как модальные окна в Runtime при клике на оборудование. Используются для детального отображения параметров узла.

Среда исполнения (Runtime)

Переключение: кнопка Runtime в заголовке.

Возможности Runtime
  • Цикл обновления: 50–1000 мс (по умолчанию 100 мс)
  • Отображаются только дисплеи и графика (логика скрыта)
  • Полноэкранный режим — F11
  • Защита доступа к редактору логином/паролем
  • Автозапуск Runtime при загрузке (опция)
  • Журнал оператора с метками времени
  • Управление слайдерами и кнопками
  • Клик по оборудованию → попап с деталями

IEC 61850 (MMS / GOOSE / SV)

MMS :102

Чтение/запись переменных IED. Управление и мониторинг.

GOOSE multicast

Быстрая передача событий (4 мс). Защитная автоматика.

SV multicast

Мгновенные значения токов трёх фаз. Цифровые подстанции.

OPC UA

OPC UA (IEC 62541) — универсальный доступ к данным. Gateway запускает сервер на opc.tcp://hostname:4840.

Настройки: SCADA → Настройки → OPC UA → endpoint, порт, режим безопасности.

Modbus TCP

Стандартный промышленный протокол. Gateway: порт 502 (или 1502 если занят). Поддерживает функции: FC 3 (Read Holding), FC 4 (Read Input), FC 6 (Write Single), FC 16 (Write Multiple).

OPC DA / InTouch / WinCC

OPC DA Classic через DCOM. Требуется Windows. Поддержка Kepware, Wonderware InTouch (Wonderware.InTouch.OPCServer), Siemens WinCC (OPCServer.WinCC).

Устранение неполадок

SCADA не подключается к Gateway

Проверьте что SCADA_Gateway.exe запущен (строка All servers running!). Убедитесь что порт 8089 не блокируется антивирусом.

Gateway: «asyncua not found»

Нормальная ситуация для автономной поставки. Gateway работает в режиме эмуляции — все данные генерируются встроенным эмулятором.

Приборы в Runtime не показывают значений

Проверьте наличие зелёной точки GW в заголовке. Если отсутствует — Gateway не подключён. Нажмите Runtime повторно после запуска Gateway.

Проект не сохраняется

localStorage переполнен. Удалите неиспользуемые проекты или экспортируйте в JSON.

Безопасность

  • Trial-режим: 2 часа работы с anti-tamper защитой
  • Целостность: SHA256 хэш JSX-кода, проверка при загрузке
  • Base64: исходный код закодирован внутри HTML
  • Защита редактора: логин/пароль в настройках
  • Роли: Оператор / Инженер / Админ
  • Пароли: SHA256 хэш в SQLite

Лицензирование

Встроенная система лицензирования без серверов активации. Каждая сборка содержит:

  • Build ID — 16-символьный SHA256 хэш
  • License Key — 4 группы по 8 символов (MD5)
  • Integrity Hash — SHA256 контрольная сумма
