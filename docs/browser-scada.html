How awtSCADA Works in the Browser

The entire SCADA engine — graphic editor, runtime, function blocks, database — is contained in a single HTML file (240 KB). It uses:

  • React 18 for the UI framework
  • HTML5 Canvas for real-time graphics rendering
  • Web Workers for parallel processing
  • IndexedDB + SQLite (sql.js) for local database
  • WebSocket for real-time communication with the gateway

Industrial Protocols

awtSCADA connects to real equipment through a Python gateway:

  • IEC 61850 — MMS, GOOSE, Sampled Values (unique for browser SCADA)
  • OPC UA — universal industrial standard
  • Modbus TCP/RTU — PLCs, sensors, meters
  • MQTT — IoT and edge devices
  • SNMP — network equipment
  • BACnet — building automation
  • DNP3 — utilities
  • IEC 60870-5-104 — power grid telecontrol
  • REST API — custom integrations
